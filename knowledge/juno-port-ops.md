# 端口占用 · Windows 处置（注入 Juno）

> 用户常同时跑「乾坤 AI」和「龙猫校园」。端口冲突不是灾难——先认人，再问杀谁，再动手。
> 启动服务绝不阻塞整轮对话：长驻命令必须后台，验端口后再说 ✅。

## 端口被占用时（必须按序）

1. **认端口属于谁**（禁止乱杀）
   ```text
   netstat -ano | findstr ":PORT"
   tasklist /FI "PID eq PID号" /FO LIST
   ```
2. **决策**（对用户说清）：杀残留 / 换端口 / 保留乾坤另开龙猫
3. **确认后** `taskkill /PID … /F`，再 `netstat` 确认空闲

## 启动前后端（禁止假报已启动）

1. `run_shell` + **正确 cwd** 跑 `pnpm dev` / `npm run dev` / `tsx watch …`（自动后台，拿 `job_id`）
2. 等 2～5 秒 → `shell_job` 看日志有没有 ready / Local: / listening
3. `netstat` 或 `curl` 验证端口真在听
4. **只有第 3 步过了** 才能对用户写 ✅ 已启动 + URL
5. 启动失败：贴日志要点 + 端口冲突选项，不要连环空转同一命令

### 禁止
- 扫到占用就停住、不给杀/换选项
- 后台 spawn 完就写「前端已启动」却不验端口（用户一眼拆穿）
- 前台死等 `pnpm dev` 把整轮对话拖到「请求超时，已解锁」
- 不问清就杀乾坤占用的端口
